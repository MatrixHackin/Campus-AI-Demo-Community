# Vibe Coding 开发手册

如果你已经有技术背景，只需要确保：`应用监听3000端口,base_path设置为“/apps/你的应用名称”`。如果你不熟悉开发流程，请继续往下看。

另外，所有用户都有一块持久化存储，挂载路径为 `/mydata`。需要长期保存的数据、代码或配置建议放在这里，避免开发沙盒重建后丢失。

这份手册面向第一次接触开发平台的同学，只要按步骤操作，就可以在平台上创建一个开发沙盒，用浏览器、VS Code、Cursor 或命令行开始开发你的应用。

应用的呈现形式有很多种：比如移动端应用(手机app比如支付宝、微信等)，桌面端应用（Word, PowerPoint等），网页端应用（通常呈现形式是可交互的网站）。网页端应用不需要额外的下载安装，只需要浏览器即可使用，本平台主要面向网页端应用的开发与上线。

开发网页端应用不可规避的过程就是Coding，但在AI Agent发展的如今，Coding已经不再局限于专属于技术人员的行为，你不需要学习复杂的编程语言或者语法，只需要用自然语言清楚的描述你的需求，Agent会为你完成剩下的一切。当然，Agent可以为你完成信息世界的任务，但没有办法直接冲破次元壁完成物理世界的任务，比如你不能让Agent帮你搬一台电脑到家里（除非有硬件帮助），也不能凭空让你在自己电脑上开发的应用可以被另一台电脑使用。因此我们会为你解决上线应用这最后一公里，你要做的就是挥洒你的创意，开始Coding吧！

## 一些概念
Vibe Coding可以通俗描述为：`用户通过自然语言描述需求，智能Agent为你完成需求`。比如你告诉Agent:`我想做一个**网页端**的贪吃蛇小游戏，Agent就会帮你做好。

有许多提供Agent服务的供应商，你或许已经听过他们的名字，你可以通过免费额度或者付费的方式来使用Agent服务，当然，Agent的工作能力高低不一，价格也不一，你需要选择适合你自己的。以下为你提供一些参考。

- DeepSeek
- Copilot
- Claude Code
- OpenAI Codex
- Composer
- Kimi

和Agent交互的主流方式有两种
- IDE/IDE插件：我们把VSCode、Cursor这些可以用于开发的工具简称为IDE(Integrated Development Environment)
- CLI: 命令行界面(Command-Line Interface)是在复杂美观的图形界面出现以前人们和计算机交互的方式（仅使用键盘输入输出与计算机进行交互）。你的电脑一般都会自带一个CLI工具，如Windows的PowerShell，或者MacOS的Terminal。

你可以把整套流程理解成：
1. 在“工作台”创建一个开发沙盒。
2. 用浏览器、开发工具（如VSCode, Cursor） 或命令行窗口连进去使用电脑。
3. 在远程开发电脑里与Agent交互、写代码、运行应用。
4. 请告诉你的agent或自行配置，确保`应用监听3000端口,base_path设置为“/apps/你的应用名称”`后，我们会为你分配网址来访问你的应用。
5. 做完后可以发布到应用市场, 让大家都可以使用你的应用。

## 从零开始完成一次开发

### 第一步：创建开发沙盒

1. 登录平台。
2. 进入“工作台”。
3. 在右侧“模板仓库”中选择一个开发环境模板。
4. 不确定选哪个时，保持默认选择 `devbox`。
5. 点击“创建开发沙盒”。
6. 填写 `app_name`。
7. 设置连接密码，这个密码将用于连接这台远程开发电脑。
8. 点击“确认创建”。
9. 等待沙盒状态变为“运行中”。

### 第二步：进入开发沙盒

开发沙盒运行后，你可以选择一种方式进入：

- CLI使用Agent:
  - 想用浏览器操作：点 `WebSSH`。
  - 想用本机命令行：点 `复制 SSH`, 进入你的电脑自带的命令行终端后粘贴并按下回车，输入连接密码后即可连接。
- IDE使用Agent:
  - 想用 VS Code 写代码：点 `VSCode连接`。
  - 想用 Cursor 写代码：点 `Cursor连接`。

## 利用 IDE 开始 Vibe Coding

IDE 就是代码编辑器。VS Code 和 Cursor 都可以通过 SSH 连接到平台开发沙盒。连接成功后，你看到的是本机编辑器界面，但文件和命令实际运行在远程开发沙盒里。

适合使用 IDE 的情况：

- 你需要长时间写代码。
- 你希望有文件树、搜索、补全、Git 面板。
- 你希望使用 Copilot、Cursor Agent 等 AI 编程助手。
- 你希望边写代码边在内置终端运行命令。

### VS Code + GitHub Copilot

VS Code 是常用代码编辑器，GitHub Copilot 是可以在 VS Code 里补全代码、解释代码和生成代码的 AI 助手。

#### 本机准备

1. 安装 VS Code：[https://code.visualstudio.com/](https://code.visualstudio.com/)
2. 在 VS Code 扩展市场安装 `Remote - SSH`。
3. 如需 AI 辅助，安装 `GitHub Copilot` 和 `GitHub Copilot Chat`。
4. 确认你的 GitHub 账号已经开通 Copilot，或学校、团队已提供授权。

#### 连接开发沙盒

1. 在平台“工作台”找到目标开发沙盒。
2. 确认沙盒状态为“运行中”。
3. 点击 `VSCode连接`。
4. 浏览器询问是否打开 VS Code 时，选择允许。
5. VS Code 要求输入密码时，输入创建开发沙盒时设置的连接密码。
6. 等待 VS Code Server 安装完成。
7. 连接成功后，打开项目目录开始开发。

第一次连接可能比平时慢，因为 VS Code 需要在开发沙盒内安装远程服务。之后再次连接通常会更快。

#### 推荐使用方式

先让 Copilot 理解项目：

```text
请阅读当前项目结构，不要修改文件。用通俗语言说明这个项目如何启动、主要目录有什么作用。
```

然后让它提出方案：

```text
我要实现一个功能：描述你的需求。请先给出最小改动方案，不要直接修改文件。
```

确认方案后再让它动手：

```text
按刚才的方案修改。修改后请告诉我改了哪些文件，以及我应该运行什么命令验证。
```

### Cursor

Cursor 是内置 AI Agent 的代码编辑器。它适合让 AI 阅读整个项目、规划修改、批量改代码。

#### 本机准备

1. 下载 Cursor：[https://cursor.com/download](https://cursor.com/download)
2. 登录 Cursor 账号。
3. 按需要开通套餐或配置可用模型。
4. 确认本机可以使用 `ssh` 命令。
5. 如果 Cursor 连接失败，先用平台的 `复制 SSH` 命令在本机终端验证能否连接。

#### 连接开发沙盒

1. 在平台“工作台”找到目标开发沙盒。
2. 点击 `Cursor连接`。
3. 浏览器询问是否打开 Cursor 时，选择允许。
4. Cursor 要求输入密码时，输入创建开发沙盒时设置的连接密码。
5. 连接成功后，在远程窗口中打开项目目录。

#### 推荐使用方式

Cursor 的 Agent 能力比较强，但也更容易一次改动很多文件。新手建议明确限制它的行为：

```text
请先只阅读项目，不要修改任何文件。总结项目结构、启动命令和最可能出问题的配置。
```

需要修改时：

```text
请用最小改动实现以下需求。每次修改前先说明计划，不要重构无关代码。
```

修改后：

```text
请检查 git diff，说明每个文件为什么要改，并给出验证命令。
```

## 利用 CLI 开始 Vibe Coding

CLI 是命令行工具。你可以在开发沙盒终端中直接运行 Claude Code、Codex CLI、npm、python、git 等命令。

适合使用 CLI 的情况：

- 你熟悉终端命令。
- 你希望 AI 直接在项目目录里读代码、改代码、跑测试。
- 你使用 WebSSH 或本机 SSH 就能完成开发。

### 进入开发沙盒终端

#### 方式一：WebSSH

WebSSH 是浏览器里的终端。

操作步骤：

1. 在“工作台”找到目标开发沙盒。
2. 点击 `WebSSH`。
3. 浏览器打开终端页面。
4. 输入创建开发沙盒时设置的连接密码。
5. 看到命令提示符后即可输入命令。

适合：

- 临时查看文件。
- 快速启动应用。
- 查看日志。
- 不方便安装本地工具时使用。

#### 方式二：复制 SSH

如果你会使用本机终端，建议使用原生 SSH。

操作步骤：

1. 在“工作台”点击 `复制 SSH`。
2. 打开本机终端。
3. 粘贴复制的命令。
4. 输入创建开发沙盒时设置的连接密码。

### Claude Code CLI

Claude Code 是 Anthropic 提供的命令行 AI 编程工具。它适合让 AI 在终端里阅读项目、解释代码、规划任务、修改文件和运行命令。

#### 安装

官方文档：[https://docs.claude.com/en/docs/claude-code/cli-reference](https://docs.claude.com/en/docs/claude-code/cli-reference)

常见安装方式：

```bash
npm install -g @anthropic-ai/claude-code
claude --version
```

如果提示没有 `npm`，说明当前环境没有 Node.js 或 npm，需要先安装 Node.js，或换用已经预装 Node.js 的开发环境模板。

#### 登录或配置

按照 Claude Code 的终端提示登录账号或配置 API 使用方式。不同账号类型、学校授权或付费方式可能不同，请以官方提示为准。

#### 使用

进入项目目录：

```bash
cd /path/to/project
```

启动 Claude Code：

```bash
claude
```

推荐第一句话：

```text
请阅读当前仓库，不要修改文件。用通俗语言说明这个项目如何启动、如何验证、有哪些风险。
```

确认理解后再让它修改：

```text
请按最小改动实现以下需求。修改前先给计划，修改后运行必要验证，并说明改动文件。
```

### Codex CLI

Codex CLI 是 OpenAI 提供的命令行 AI 编程工具。它适合在终端中进行代码阅读、补丁修改、运行测试和提交前检查。

#### 安装

项目地址：[https://github.com/openai/codex](https://github.com/openai/codex)

常见安装方式：

```bash
npm install -g @openai/codex
codex --version
```

#### 配置 OpenAI API

OpenAI API 快速开始文档：[https://platform.openai.com/docs/quickstart](https://platform.openai.com/docs/quickstart)

创建 API Key 后，在终端中设置环境变量：

```bash
export OPENAI_API_KEY="你的 API Key"
```

然后进入项目目录运行：

```bash
codex
```

#### 使用 OpenAI 兼容平台

有些平台提供 OpenAI 兼容接口，通常会给你三样信息：

- API Key：身份凭证。
- Base URL：接口地址。
- Model：模型名称。

你需要按照对应 CLI 或 SDK 的配置方式填写这些信息。

国内常见选择包括：

- DeepSeek 平台：[https://platform.deepseek.com/](https://platform.deepseek.com/)
- 阿里云百炼：[https://bailian.console.aliyun.com/](https://bailian.console.aliyun.com/)

各平台价格、模型名和额度会变化，请以对应官方控制台为准。

#### 推荐提示词

第一次让 Codex 看项目：

```text
请先检查当前项目，不要修改文件。总结项目结构、启动命令、测试命令和潜在风险。
```

让 Codex 修改代码：

```text
请用最小改动实现以下需求。不要重构无关代码。修改后运行构建或测试，并说明结果。
```

让 Codex 检查改动：

```text
请检查当前 git diff，找出可能的 bug、安全风险和可以简化的地方，不要直接修改文件。
```

## 如何和 AI 说明需求

### 好需求怎么写

好的需求应该包含三部分：

1. 你想实现什么。
2. 当前哪里不符合预期。
3. 你希望验证结果是什么。

示例：

```text
我想在工作台开发沙盒列表里新增一个“复制链接”按钮。
点击后复制当前应用访问地址。
请保持现有样式一致，修改后运行前端 build 验证。
```

### 不建议的写法

不建议只写：

```text
帮我优化一下。
```

原因是 AI 不知道你想优化速度、界面、代码结构还是用户体验，容易做出不必要的大改动。

### 推荐让 AI 先计划

如果需求比较复杂，先让 AI 给计划：

```text
请先不要改代码。先阅读相关文件，给出实现计划、影响范围和验证方式。
```

你确认计划后，再让它实现。

### 修改后一定要验证

常见验证命令：

```bash
npm run build
```

```bash
python -m compileall -q app
```

```bash
git diff --check
```

不同项目验证命令不同。如果你不确定，可以问 AI：

```text
这个项目修改后应该运行哪些验证命令？请先说明，不要执行危险命令。
```

## 发布应用到应用市场

### 发布前检查

发布前请确认：

- 沙盒状态是“运行中”。
- 应用能通过 `访问应用` 打开。
- 页面样式和按钮正常。
- 没有把 API Key、密码或个人隐私显示在页面上。
- 应用描述简单清楚。

### 发布步骤

1. 在“工作台”找到目标开发沙盒。
2. 点击 `发布`。
3. 填写应用简述。
4. 可选上传封面图。
5. 点击确认。
6. 发布后应用会出现在“应用市场”。

### 应用描述怎么写

建议用一句话说明应用能做什么。

好的例子：

```text
一个帮助学生整理课程笔记并生成复习提纲的工具。
```

不好的例子：

```text
很好用的系统。
```

### 封面图片建议

- 使用清晰截图或简洁图标。
- 不要上传含敏感信息的图片。
- 不要上传过于复杂的大图。
- 平台会压缩封面，但你仍应尽量选择轻量图片。

## 保存为我的模板

如果你在开发沙盒里安装了依赖、配置了环境，想下次继续使用，可以点击 `保存为我的模板`。

“保存为我的模板”可以理解为：把当前开发沙盒打包成一个新的开发环境模板。

适合保存的情况：

- 你已经安装了很多依赖。
- 你配置好了开发环境。
- 你希望下次不用重新安装。

不建议保存的情况：

- 开发沙盒里保存了密码、API Key 或私密文件。
- 项目还没有整理好。
- 只是临时测试。

保存完成后，新的开发环境模板会出现在“我的模板”中。之后创建开发沙盒时可以选择它。

## 安全规范

### 不要泄露密钥

不要把以下内容写进代码仓库：

- API Key。
- 数据库密码。
- SSH 密码。
- Cookie。
- Token。
- `.env` 中的生产配置。

如果 AI 要求你粘贴这些内容，应该拒绝。

### 使用 .env 管理配置

推荐把密钥放在 `.env` 文件里，并把 `.env` 加入 `.gitignore`。

推荐 `.gitignore`：

```gitignore
.env
.env.*
!.env.example
*.key
*.pem
```

可以提供一个 `.env.example`，只写变量名，不写真正密钥：

```text
OPENAI_API_KEY=
DATABASE_URL=
```

### 修改前先保存现场

在让 AI 大量修改之前，建议先运行：

```bash
git status
```

如果当前改动重要，先提交或备份。

这样如果 AI 改坏了，可以回退。

### 人工检查 AI 改动

AI 修改后，不要直接上线。至少检查：

```bash
git diff
```

重点看：

- 有没有删掉不该删的代码。
- 有没有写死密码或地址。
- 有没有改动无关文件。
- 有没有引入明显不安全逻辑。

## 排障清单

### 创建开发沙盒失败

检查：

- `app_name` 是否符合规则。
- `app_name` 是否已经被使用。
- 是否选择了开发环境模板。
- 连接密码是否至少 6 位。
- 稍后刷新页面再试一次。

### 开发沙盒一直创建中

可能原因：

- 开发环境模板拉取较慢。
- 集群资源暂时紧张。
- 开发环境模板本身有问题。

建议：

- 等待一两分钟。
- 刷新工作台。
- 如果一直不成功，换回默认 `devbox` 开发环境模板再试。

### IDE 按钮点击后没有反应

检查：

- 本机是否安装 VS Code 或 Cursor。
- 浏览器是否拦截了自定义协议跳转。
- VS Code 是否安装了 Remote - SSH。
- Cursor 是否支持当前 SSH 连接方式。
- 先点击 `复制 SSH`，在本机终端确认 SSH 能连通。

### SSH 密码错误

检查：

- 密码是否是创建开发沙盒时设置的连接密码。
- 是否连接到了正确的 `app_name`。
- 开发沙盒是否仍然是“运行中”。
- 是否复制了旧开发沙盒的 SSH 命令。

### VS Code 或 Cursor 卡在安装 Server

第一次连接时，编辑器会在开发沙盒里安装远程服务，可能需要一些时间。

可以尝试：

- 等待一会儿。
- 关闭远程窗口后重新连接。
- 用 `复制 SSH` 在终端确认网络和密码正常。
- 检查开发沙盒磁盘空间是否不足。

### WebSSH 卡住或显示不刷新

可以尝试：

- 刷新 WebSSH 页面。
- 关闭后重新打开。
- 避免在 WebSSH 中长时间运行大量交互输出的命令。
- 长时间开发建议使用本机 SSH、VS Code 或 Cursor。

### 应用访问 Gateway Timeout

检查：

- 应用是否已经启动。
- 应用是否监听 `3000` 端口。
- 是否监听 `0.0.0.0`，而不是只监听 `127.0.0.1`。
- 开发沙盒是否仍然运行。

常见正确启动方式：

```bash
npm run dev -- --host 0.0.0.0 --port 3000
```

### 页面能打开但样式丢失

基本原因是 Base Path 没配置好。

请检查：

- Vite 是否设置 `base: '/apps/{app_name}/'`。
- Next.js 是否设置 `basePath`。
- React Router 是否设置 `basename`。
- 静态资源路径是否错误地以 `/assets` 开头。

### API Key 无效

检查：

- Key 是否复制完整。
- 是否多了空格、换行或引号。
- 是否使用了错误平台的 Key。
- 是否已开通模型权限。
- 是否已经充值或分配额度。
- Base URL 是否和 Key 属于同一个平台。
